# WORKFLOW PRINCIPLES

기준일: 2026-03-21

이 문서는 `danal-research`의 분석 운영 원칙과 내부 구조 레퍼런스를 정리한 메모다.

## 1. 구조적으로 어느 쪽에 더 가까운가

- 결론:
  - `danal-research`는 현재 기준 `eimas`보다 `eco_system_v2`에 더 가깝다.
- 이유:
  - 범용 투자 자동화보다 `workflow-first` 성격이 강하다.
  - `신호 수집 -> 레짐/시나리오 판단 -> 보고서/차트 handoff` 구조가 분명하다.
  - `confidence`, `반증 가능 신호`, `watch point`, `사람 검토 전제`를 남기는 방식이 중심이다.
- 차이:
  - `eco_system_v2`는 거시 신호 합의와 자산 판단 시스템에 가깝다.
  - `danal-research`는 그 운영 원칙을 다날용 `brief / IM / screen / chart` 문서 흐름으로 좁힌 vertical이다.

## 2. 실제로 녹여둔 원칙

### 2.1 Workflow First

- 기능 데모보다 `내부 문서 생산 흐름`을 먼저 둔다.
- 출력 중심은 `brief`, `IM`, `screen`, `master report`, `chart`, `CSV`, `Excel`이다.

### 2.2 Deterministic Core First

- 데이터 수집, 지표 계산, 레짐 판단, 시나리오 표는 정형 계산과 규칙으로 고정한다.
- 자연어는 그 결과를 사람이 읽는 문서로 구조화하는 단계에 집중한다.

### 2.3 Confidence and Falsification

- 결론만 적지 않는다.
- `확신도`, `반론`, `반증 가능 신호`, `다음 주 watch point`를 같이 남긴다.
- 이 방식은 다날이 원하는 `내부 판단 보조 문서`와 더 잘 맞는다.

### 2.4 Human-in-the-Loop

- 인턴 역할은 초안 작성과 자료 정리다.
- 최종 투자 판단과 실행은 시니어 검토를 전제로 둔다.
- 그래서 산출물도 `판단 보조 자료` 톤을 유지한다.

### 2.5 Action-Oriented Handoff

- 모든 결과는 `다날에 어떤 의미가 있는가`로 닫는다.
- 즉 분석의 끝을 일반 시장 코멘트가 아니라 `결제 본업`, `가맹점 가치`, `글로벌/외국인 결제`, `디지털자산 확장` 중 하나로 연결한다.

### 2.6 Harness Retained

- 역할 분리 하니스, 품질 게이트, 스킬 주입 구조는 유지한다.
- 다만 외부 설명에서는 그것을 주인공으로 두지 않고 `반복 실행 안정성`과 `문서 품질 관리`를 돕는 보조 레이어로 위치시킨다.
- 즉 구조는 남기고, 서사만 다날의 금융·투자 분석 직무에 맞게 재배치한다.

## 3. 일부러 가져오지 않은 것

- 범용 자산군 전체를 다루는 넓은 서사
- 실시간 운용·백테스트·포트폴리오 실행 중심 메시지
- 멀티 에이전트 자체를 주인공으로 두는 설명
- 대시보드/플랫폼 완성도보다 앞서는 시스템 자랑

## 4. 지원 포지션 기준으로 번역하면

- 이 프로젝트는 `다날 금융·투자 분석 인턴`이 실제로 맡을 법한 문서 업무를 좁고 깊게 구현한 포트폴리오다.
- 강점은 `AI를 썼다`가 아니라 `시장/기업/거시 자료를 내부 문서, 차트, 시나리오 테이블로 바꿨다`는 데 있다.
- 설명할 때는 레퍼런스 프로젝트 이름보다 `workflow first`, `deterministic core`, `confidence + watch point`, `human review` 원칙을 말하는 편이 더 낫다.
